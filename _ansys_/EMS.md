# 智慧园区综合能源管理系统（EMS）分层算法选型+特性+选用理由总结
整体架构自上而下分为**应用交互层、决策优化层、预测感知层、设备底层拟合层、数据预处理层**，核心原则：**数值计算、约束优化、工况拟合全部用小参数机器学习/运筹优化；仅上层自然语言、非结构化数据处理用轻量化大模型（7B及以内量化）**，严禁大模型接管能源闭环量化决策。

## 一、五层完整算法选型总览
### 第一层：应用交互层（业务展示、人机交互、文档输出）
#### 选用算法：轻量化开源大模型（Qwen-7B、Llama3-8B 4/8bit量化）
#### 核心特性
1. 擅长文本理解、结构化数据转自然语言、文档生成、经验知识抽取；
2. 输出具备语义解释能力，可对接运维人员自然语言问答；
3. 数值输出不稳定、带随机性，无法严格绑定设备安全约束；
4. 推理延迟高，不能用于秒级、分钟级闭环控制。
#### 选用理由
1. 承接EMS后端量化结果，自动生成节能报告、碳排放月报、改造可研、运维分析文案；
2. 解析非结构化数据：巡检工单、设备说明书、老旧运维日志、设计院改造方案，提炼人工调度经验转为系统规则；
3. 自然语言指令转标准化调度需求（如“商业区晚高峰节能5%”转为约束参数下发下层）；
4. 仅做展示与解读，不参与能耗、出力、负荷的核心计算，规避数值失真、合规不可追溯风险。

### 第二层：决策优化层（EMS核心调度、技改仿真、多目标优化、容量配置）
#### 选用算法：运筹优化（LP/MILP）+ 轻量强化学习（浅层PPO/DQN，参数＜10万）
#### 核心特性
1. 运筹优化：严格硬约束求解（机组出力上限、冷热供需平衡、管网压力、电价、碳排放限额），结果唯一、可复现、可审计；
2. 轻量强化学习：浅层全连接网络，适配分时电价、需求响应、储能充放电动态博弈，适配柔性负荷实时协同；
3. 设备插拔友好：新增/淘汰设备仅增减约束变量，无需整体重训；
4. 离线可批量遍历上百套改造方案，测算回本周期、节能率最优配比。
#### 选用理由
1. 园区冷热电联供属于带约束工程优化问题，数学规划是工业EMS标准落地方案，满足电网、节能验收审计要求；
2. 强化学习负责动态自适应场景：峰谷电价套利、电网需求响应邀约、季节性设备投切，弥补静态规划无法应对动态波动的短板；
3. 完全适配设备增减：新增热泵、储能、光伏仅扩展决策变量，原有优化框架不动，扩展性极强；
4. 算力需求极低，园区本地服务器离线批量仿真效率极高。

### 第三层：预测感知层（冷热电气负荷预测、气象耦合、中长期扩容预判）
#### 选用算法：TFT、LightGBM/XGBoost时序版、GRU/TCN（小参数时序模型）
#### 核心特性
1. 多特征融合：气温、节假日、入驻率、分时电价、业态类型、历史工况一体化建模；
2. 可解释性强：输出特征重要度，定位负荷波动核心诱因（气温/人流/电价）；
3. 增量学习友好：新增楼栋、新增用能设备仅追加特征列+少量样本微调；
4. 短/中长期全覆盖：15min短时预测、日级调度预测、月度/年度扩容负荷预判。
#### 选用理由
1. 结构化时序工业数据最优适配，远优于大模型时序预测精度与稳定性；
2. 预测结果直接喂给上层优化器作为供需边界，数值稳定是调度安全前提；
3. 园区扩建、业态招商、设备扩容时，仅需补充新增区域运行数据即可更新预测，迭代成本极低；
4. 边缘端可部署，支持离线预测，数据不出园区满足能源数据保密要求。

### 第四层：设备底层拟合层（单台设备效率建模、老化衰减、出力特性）
#### 选用算法：XGBoost、CatBoost、浅层MLP（数千~数万参数）
#### 核心特性
1. 精准拟合机组**负荷率-能耗-效率**非线性曲线（燃气轮机、冷水机组、余热锅炉、热泵、储能）；
2. 单设备独立子模型架构，模块化插拔；
3. 可自适应设备老化：长期运行效率衰减可通过增量数据自动修正；
4. 输入设备额定参数、进出口温压、流量即可输出精准能耗损耗。
#### 选用理由
1. 每台增减设备独立训练子模型，新设备试运行7~15天数据即可收敛，接入主EMS无耦合冲突；
2. 替代复杂机理建模，无需搭建精细热力学方程，工程落地成本远低于机理模型；
3. 拟合结果作为优化层的设备能耗输入，保障调度仿真贴合真实物理工况；
4. 故障低效识别：偏离标准效率曲线即可判定设备异常，联动运维预警。

### 第五层：数据预处理层（IoT清洗、补全、异常检测、数据归一）
#### 选用算法：孤立森林、One-Class SVM、KNN、线性插值、简单统计规则
#### 核心特性
1. 轻量化、极速运算，适配海量传感器秒级数据流清洗；
2. 识别传感器断线、数值跳变、管道泄漏、机组异常低效；
3. 低成本补齐缺失测点数据，保障上层预测、优化数据连续性。
#### 选用理由
1. EMS底层IoT测点量大、噪声高，轻量异常检测模型性价比最高；
2. 为上层所有算法提供干净可信数据源，是整套EMS稳定运行基础；
3. 设备增减带来测点新增时，规则化适配即可，无需重新训练复杂模型。

## 二、关键边界硬性区分（什么时候绝对不能混用）
1. **大模型禁止场景**：负荷预测、设备能耗计算、机组出力分配、技改收益测算、碳排放核算、需求响应上报（随机性导致不合规、不可审计）；
2. **小参数机器学习/运筹优化禁止场景**：非结构化文本解读、运维问答、报告生成、老旧纸质经验数字化（无自然语言理解能力）；
3. **设备增减统一适配逻辑**：底层设备子模型增量更新→预测层新增特征微调→优化层增减约束变量→上层大模型仅更新知识库文本，全程解耦，无全局重训负担。

## 三、极简一句话终极总结
1. **底层量化闭环（拟合、预测、优化、清洗）**：全程小参数机器学习+运筹优化，主打稳定、约束合规、可审计、易扩展、适配设备增减；
2. **上层业务赋能（交互、文档、经验萃取）**：仅用7B级以内轻量化量化大模型做辅助，只解读不决策、只整理不算数；
3. 智慧园区EMS智能化升级，是**分层能力叠加**，绝非用大模型替换传统工业能源算法。
