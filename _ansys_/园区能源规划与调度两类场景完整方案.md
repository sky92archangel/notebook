# 园区能源规划与调度两类场景完整方案

# 分两类场景完整方案：新建园区容量规划 / 已建成园区实时调度

核心区分：

1. **新建园区（规划层）**：决策变量 = 设备采购台数 / 额定容量（离散 \+ 连续混合变量）；目标 = 全生命周期成本、外购电 / 冷热最小、富余能量外送收益、碳排放；**离线多目标帕累托寻优**，对实时性无要求，追求方案全局最优、解集完整；

2. **已建成园区（运行调度层）**：设备容量固定，决策变量 = 逐时机组出力、储能充放、购售能功率；约束 = 设备上下限、分时电价 / 热价、实时负荷波动；**在线秒级 / 分钟级滚动优化**，强时序、强不确定性，需要快速响应实时工况。

## 一、场景 1：新建未投运园区 —— 设备采购容量规划（目标：外购最少、尽量外送能量）

### 核心需求

变量含**离散台数（几台微燃机、几台溴化锂）\+ 连续额定容量（储冷 / 储热体积）**；输入仅全年预估负荷、设备样本参数、气价电价基准；输出一组帕累托方案（投资高但外购极少、可对外售能；投资低但外购多），用于方案比选。

### 推荐算法分三档（从工程落地→深度学习加速前沿）

#### 1）工程首选：MOEA/D（替代 NSGA\-II，优先推荐）

- 优势：分解多目标，收敛速度、解集均匀性全面优于 NSGA\-II；3 目标及以上（外购能量、总投资、碳排）生成帕累托超曲面质量极高；天然适配混合整数（台数 \+ 容量）变量；能源规划领域标准算法。

- 配套双层框架（行业标准）：

    - 上层 MOEA/D：优化设备采购数量、额定容量；

    - 下层 MILP 混合整数线性规划：对每组容量方案做全年时序运行仿真，计算外购 / 外送能量、年化收益，作为上层目标函数输入。

- 适用：园区规模中等、设备种类≤10 种，纯机理仿真，无深度学习也可完成。

#### 2）高维 / 多场景加速方案：MLP 代理模型 \+ MOEA/D（PyTorch 实现）

当园区变量极多（多储能、光伏、多型号机组）、全年时序仿真单次耗时几分钟，迭代几十万次不可行时用深度学习提速：

1. 拉丁超立方 LHS 批量采样设备容量方案；

2. PyTorch 搭建**多输出 MLP（带 BatchNorm）**，输入：各设备容量 / 台数；输出：年外购电量、年外送收益、总投资、碳排放；

3. 训练完成后，MOEA/D 直接调用神经网络替代时序仿真，计算速度提升几十倍；

4. 最优帕累托解回代原始机理模型校验误差。

- 网络选型：只用基础 MLP 即可，无需复杂 GAN/Transformer；无实时在线需求，监督学习足够。

#### 3）高精度小规模园区：ε\- 约束法（确定性数学规划）

变量少（3\~5 种设备）、需要精准无随机误差帕累托前沿时使用；把 “外购能量最小” 作为主目标，其余目标设约束区间遍历求解，适合小型产业园可研报告精确计算。

#### 不推荐方案

- NSGA\-II：高维多目标收敛差、解集分布不均，三联供规划现在基本淘汰；

- GAN / 强化学习：规划是静态离线选型，无序列决策，强化学习完全不适用；

- 多目标 PSO：混合离散台数变量编码麻烦，稳定性不如 MOEA/D。

### 规划层完整落地流程

1. 建立全年逐时冷热电负荷时序模型；

2. 搭建 CCHP 机理 MILP 运行仿真（下层）；

3. 上层 MOEA/D 迭代寻优，生成帕累托超曲面；

4. 用 HV、IGD 评价解集，筛选 “外购最低、外送收益最高” 候选方案；

5. 如需批量测算多负荷工况，用 PyTorch MLP 代理加速。

## 二、场景 2：已建成园区实时调控调度（设备固定，分时能源价格、实时负荷波动）

### 核心需求

变量全是**连续时序出力**（燃气轮机功率、溴化锂制冷量、储能充放、购电 / 售电功率）；每 5/15 分钟滚动更新负荷、电价；约束严格：机组出力上下限、储能 SOC 边界、冷热供需平衡；目标：实时外购能量最小、富余能量最大化对外售卖、运行成本最低；**必须毫秒 / 秒级输出控制指令**。

### 推荐算法分三档（传统滚动优化 → PyTorch 深度强化学习）

#### 1）传统工业成熟方案：MILP 滚动时域优化（基准方案）

- 逻辑：每 15 分钟滚动更新未来 24h 负荷 / 电价预测，求解混合整数线性规划，输出未来调度出力；

- 优点：机理精准、约束严格满足、可直接对接 DCS 控制系统；

- 缺点：单次求解数十秒，负荷突变、多储能耦合时计算卡顿，无法做到纯实时；

- 适用：调度平台后台日前优化、慢尺度日内调度。

#### 2）实时在线最优方案：多目标深度强化学习 MORL（PyTorch 主推）

完美匹配**时序连续决策、实时交互、价格 / 负荷不确定性**，训练完成后推理仅几毫秒，真正实时调控，完全贴合 “最小外购、最大化外送售能” 目标。

##### 分两种 PyTorch 网络选型

1. **PPO\-MORL（最稳、工程落地首选）**

    - 动作空间：连续变量（机组出力、储能充放、购售能功率）；

    - 状态空间：实时冷热电负荷、储能 SOC、当前分时电价 / 热价、机组剩余可调裕度；

    - 奖励函数设计（贴合你的需求）：
    $R = -W_1 \cdot E_{buy} + W_2 \cdot E_{sell} - W_3 \cdot Cost_{gas} - Penalty$
    $E_{buy}$ 外购能量、$E_{sell}$ 对外售卖能量、惩罚项用于设备越限、冷热不平衡；

    - 优势：训练稳定、超参数易调、鲁棒性强，负荷剧烈波动不会失控，国内园区综合能源调度大量落地案例。

2. **SAC\-MORL（追求最优收益，适合电价波动大、频繁外送）**
软 Actor\-Critic，最大熵策略，能充分捕捉峰谷价差套利，富余能量最大化上网售卖；适合园区有稳定外网售电 / 售热通道场景。

##### 不推荐 DQN

DQN 只能离散动作，机组出力分段离散会产生控制误差，连续出力场景优先 Actor\-Critic 类（PPO/SAC/DDPG）。

#### 3）混合方案：代理模型 \+ 滚动优化（折中路线）

如果不想完全放弃机理、又想提速：

- PyTorch 训练时序 Transformer/LSTM 代理模型，拟合 CCHP 逐时供需平衡；

- 滚动优化时调用神经网络替代机理仿真，缩短求解时间；

- 适合已有成熟调度平台，仅做提速改造，不重构控制逻辑。

#### 不推荐方案

- NSGA\-II/MOEA/D 等进化算法：单次迭代数十分钟，完全无法满足实时调度；

- ParetoGAN/CVAE：仅能批量生成静态方案，不能实时响应分钟级负荷、电价变化。

### 调度层落地流程（PyTorch DRL 路线）

1. 采集全年历史负荷、电价、设备运行数据构建环境模拟器；

2. PyTorch 搭建 PPO/SAC 多目标强化学习智能体；

3. 离线预训练上万轮，优化外购最小、外送收益最大策略；

4. 部署在线推理模块，每 5 分钟输入实时工况，毫秒输出设备调控指令；

5. 滚动更新数据集，周期性重训练适配季节负荷变化。
